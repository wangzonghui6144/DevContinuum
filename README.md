# English Version

## 💡 Background Story

Have you ever experienced these development scenarios:

🤔 **"This feature is so complex, I forget the original design halfway through..."**

😵 **"The task is too big, the AI assistant gets sidetracked and loses key information..."**

😰 **"The project structure is too complex, I have to re-explain the tech stack and standards every time..."**

🤯 **"The system crashes halfway through development and all progress is lost..."**

**DevContinuum was born to solve these pain points!**

## 🎯 What Problems Do We Solve?

### 1. 🧠 Long Context Loss Problem
- **Problem**: AI easily forgets early design decisions and context in large task development
- **Solution**: Automatically persist plans, progress, and decisions to project documentation
- **Effect**: Seamless recovery even after session interruption, critical information never lost

### 2. 📋 Development Process Chaos Problem
- **Problem**: Unclear development steps, easy to miss important环节
- **Solution**: Standardized Feat/Fix/CI workflows with clear guidance for each step
- **Effect**: Well-organized development process, guaranteed quality improvement!

### 3. 🔄 Progress Tracking Difficulty Problem
- **Problem**: Don't know development progress, what's completed and what's not
- **Solution**: Real-time progress synchronization to documentation, task list management
- **Effect**: Project status crystal clear, more efficient team collaboration

### 4. 🛡️ Disaster Recovery Problem
- **Problem**: Development interruption, progress loss, need to restart
- **Solution**: Checkpoint mechanism + breakpoint resume + exception handling
- **Effect**: System crashes don't matter, can resume work anytime

### 5. 📚 Knowledge沉淀 Problem
- **Problem**: Development experience cannot be effectively沉淀, need to重新摸索 next time
- **Solution**: Automatically record decision processes and lessons learned
- **Effect**: Form team knowledge base, experience reusable

## ✨ Core Features

### 🎪 Intelligent Task Planning
```
You: "Help me develop a user management system"
DevContinuum: Auto-generate detailed plan → Break down tasks → Assess risks → Set milestones
```

### 💾 Persistent Storage
- 📝 Plan documents auto-saved to `docs/plan/`
- 🏗️ Project structure documents saved to `docs/structure/`
- 📊 Progress status real-time synchronization
- 🔄 Checkpoint auto-saves execution state

### 🛠️ Standardized Workflows
- 🚀 **Feat Flow**: Standard process for new feature development
- 🐛 **Fix Flow**: Standard process for bug fixes
- 🔄 **CI Flow**: Standard process for continuous integration
- 📋 **Plan Flow**: Standard process for project planning

### 🤖 External Skill Integration
- 🔧 **Super Developer**: Complex task breakdown
- 🎭 **Midscene.js**: Browser automation testing
- 🧪 **OpenTest**: Full-process test automation
- 🎨 **Frontend Design**: UI component design

### 📈 Real-time Progress Management
- 📋 Task list-driven development
- 📊 Progress visualization tracking
- 🎯 Milestone management
- 🚨 Risk warning

## 🚀 How to Use?

### Using in Claude Code

1. **Install DevContinuum Skill**
   ```bash
   # Put DevContinuum Skill into your project
   cp -r .claude/skills/DevContinuum /your-project/.claude/skills/
   ```

2. **Trigger the Skill**
   Use these keywords in Claude Code:
   ```
   "Help me develop a..."
   "Implement... feature"
   "Fix this bug"
   "Create a development plan"
   "Start a new project"
   ```

3. **Enjoy Intelligent Assistance**
   - 📋 Auto-generate development plans
   - 🏗️ Analyze project structure
   - 🔄 Guide development process
   - 📊 Sync progress status

### Using in Cursor

1. **Configure Claude Integration**
   - Ensure Cursor is configured with Claude API
   - Use DevContinuum Skill documents as project knowledge base

2. **Reference Project Documents**
   ```
   Please refer to docs/structure/project-analysis.md for project structure
   Please refer to docs/plan/example-user-auth.md for development plan format
   ```

3. **Combine Usage**
   - Use plans generated by DevContinuum to guide development
   - Refer to standardized workflow documents
   - Follow project coding standards

### Using in Other AI Tools

**General Method**:
1. Use `docs/` directory as project documentation library
2. Read related documents before development to understand standards
3. Use `docs/plan/template.md` to create development plans
4. Follow workflows in `docs/structure/development-workflows.md`

## 📋 Quick Start Example

### Scenario: Developing User Authentication System

1. **Trigger Skill**
   ```
   You: "Help me develop a user authentication system including registration, login, password reset features"
   ```

2. **Auto-generate Plan**
   ```markdown
   # User Authentication System Development Plan
   
   ## Task Overview
   - Task Type: feat
   - Priority: High
   - Expected Completion: 5 days
   
   ## Execution Plan
   - Phase 1: Environment Setup (1 day)
   - Phase 2: Core Development (3 days)
   - Phase 3: Testing & Validation (1 day)
   ```

3. **Guide Development Process**
   ```
   DevContinuum: "First, let's analyze the project structure..."
   DevContinuum: "Next, configure the development environment..."
   DevContinuum: "Now let's start implementing user registration feature..."
   ```

4. **Real-time Progress Sync**
   ```markdown
   ## Progress Tracking
   - ✅ Environment Setup Complete (100%)
   - 🔄 Core Development In Progress (60%)
   - ⏳ Testing & Validation Pending (0%)
   ```
