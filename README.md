# DevContinuum - AI 驱动的开发流程优化工具 🚀

## 💡 背景故事

你是否经历过这样的开发场景：

🤔 **"这个功能好复杂，我写到一半就忘了最开始的设计思路..."**

😵 **"任务太大，AI 助手说着说着就跑偏了，关键信息都丢了..."**

😰 **"项目结构太复杂，每次都要重新解释一遍技术栈和规范..."**

🤯 **"开发到一半系统崩溃了，之前的进度全没了..."**

**DevContinuum 就是为了解决这些痛点而生的！**

## 🎯 我们能解决什么问题？

### 1. 🧠 长上下文丢失问题
- **问题**: 大任务开发中，AI 容易忘记早期的设计决策和上下文
- **解决方案**: 自动将计划、进度、决策持久化到项目文档中
- **效果**: 即使会话中断，也能无缝恢复，关键信息永不丢失

### 2. 📋 开发流程混乱问题  
- **问题**: 开发步骤不清晰，容易遗漏重要环节
- **解决方案**: 标准化的 Feat/Fix/CI 工作流，每一步都有明确指导
- **效果**: 开发过程井井有条，质量保证up up！

### 3. 🔄 进度跟踪困难问题
- **问题**: 不知道开发进展到哪了，哪些完成了哪些还没做
- **解决方案**: 实时同步进度到文档，任务清单式管理
- **效果**: 项目状态一目了然，团队协作更高效

### 4. 🛡️ 容灾恢复问题
- **问题**: 开发过程中断，进度丢失，需要重新开始
- **解决方案**: Checkpoint 机制 + 断点续传 + 异常处理
- **效果**: 系统崩溃也不怕，随时可以恢复工作

### 5. 📚 知识沉淀问题
- **问题**: 开发经验无法有效沉淀，下次还要重新摸索
- **解决方案**: 自动记录决策过程和经验教训
- **效果**: 形成团队知识库，经验可复用

## 🚀 如何使用？

### 在 Claude Code 中使用

1. **安装 DevContinuum Skill**
   ```bash
   # 将 DevContinuum Skill 放入项目
   cp -r .claude/skills/DevContinuum /your-project/.claude/skills/
   ```

2. **触发 Skill**
   在 Claude Code 中使用以下关键词触发：
   ```
   "帮我开发一个..."
   "实现...功能"
   "修复这个bug"
   "制定开发计划"
   "开始新项目"
   ```

3. **享受智能辅助**
   - 📋 自动生成开发计划
   - 🏗️ 分析项目结构
   - 🔄 指导开发流程
   - 📊 同步进度状态

### 在 Cursor 中使用

1. **配置 Claude 集成**
   - 确保 Cursor 已配置 Claude API
   - 将 DevContinuum Skill 文档作为项目知识库

2. **引用项目文档**
   ```
   请参考 docs/structure/project-analysis.md 了解项目结构
   请参考 docs/plan/example-user-auth.md 了解开发计划格式
   ```

3. **结合使用**
   - 使用 DevContinuum 生成的计划指导开发
   - 参考标准化的工作流文档
   - 遵循项目代码规范

### 在其他 AI 工具中使用

**通用方法**:
1. 将 `docs/` 目录作为项目文档库
2. 在开发前阅读相关文档了解规范
3. 使用 `docs/plan/template.md` 制定开发计划
4. 遵循 `docs/structure/development-workflows.md` 中的流程


## 🎯 最佳实践

### 1. 📝 计划先行
- **不要直接开始编码**，先让 DevContinuum 帮你制定计划
- **明确任务范围**，避免范围蔓延
- **识别风险**，提前准备应对措施

### 2. 🔄 小步快跑
- **将大任务拆解**为小的、可管理的子任务
- **每完成3-5个子任务**就进行一次回顾
- **及时更新进度**，保持状态同步

### 3. 📚 文档为王
- **所有重要决策**都要记录到文档中
- **定期更新项目结构文档**，保持最新状态
- **沉淀经验教训**，形成团队知识库

### 4. 🛡️ 安全第一
- **重要操作前**创建 Checkpoint
- **启用回滚机制**，确保可以安全回退
- **遵循安全规范**，避免引入漏洞

### 5. 🤝 团队协作
- **共享计划文档**，确保团队对齐
- **定期同步进度**，及时发现阻塞
- **统一代码规范**，保持代码一致性

## 🚀 开始使用

### 前置条件
- GitHub 项目
- Claude Code 或兼容的 AI 编程工具
- 基本的 Git 和开发环境

### 安装步骤
1. 克隆或下载 DevContinuum Skill
2. 将 Skill 文件放入项目的 `.claude/skills/` 目录
3. 确保项目文档结构符合规范

### 推荐配置
```yaml
# .claude/settings.json
{
  "default_skill": "DevContinuum",
  "auto_save": true,
  "checkpoint_interval": 3,
  "enable_external_skills": true
}
```
