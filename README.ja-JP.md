## 💡 背景ストーリー

このような開発シナリオを経験したことはありますか：

🤔 **"この機能は複雑すぎて、途中で最初の設計思想を忘れてしまう..."**

😵 **"タスクが大きすぎて、AIアシスタントが脱線し、重要な情報を失ってしまう..."**

😰 **"プロジェクト構造が複雑すぎて、毎回技術スタックと規約を再説明する必要がある..."**

🤯 **"開発の途中でシステムがクラッシュし、これまでの進捗がすべて失われる..."**

**DevContinuumはこれらの課題を解決するために生まれました！**

## 🎯 私たちは何を解決しますか？

### 1. 🧠 長文脈喪失の問題
- **問題**: 大規模タスク開発で、AIが初期の設計決定と文脈を忘れやすい
- **解決策**: 計画、進捗、決定をプロジェクトドキュメントに自動永続化
- **効果**: セッション中断後もシームレスに回復、重要情報が決して失われない

### 2. 📋 開発プロセスの混乱問題
- **問題**: 開発ステップが不明確で、重要な環節を見落としやすい
- **解決策**: 標準化されたFeat/Fix/CIワークフローで各ステップに明確なガイダンス
- **効果**: 整理された開発プロセス、品質保証の向上！

### 3. 🔄 進捗追跡の困難な問題
- **問題**: 開発進捗がわからない、どれが完了してどれがまだか
- **解決策**: ドキュメントへのリアルタイム進捗同期、タスクリスト管理
- **効果**: プロジェクト状態が一目瞭然、チームコラボレーションがより効率的に

### 4. 🛡️ 災害復旧の問題
- **問題**: 開発中断、進捗喪失、やり直しが必要
- **解決策**: チェックポイントメカニズム + ブレークポイント再開 + 例外処理
- **効果**: システムクラッシュでも大丈夫、いつでも作業を再開可能

### 5. 📚 知識沈降の問題
- **問題**: 開発経験が効果的に沈降せず、次回も再摸索が必要
- **解決策**: 決定プロセスと教訓を自動記録
- **効果**: チーム知識ベースを形成、経験を再利用可能

## 🚀 使用方法？

### Claude Codeでの使用

1. **DevContinuum Skillのインストール**
   ```bash
   # DevContinuum Skillをプロジェクトに配置
   cp -r .claude/skills/DevContinuum /your-project/.claude/skills/
   ```

2. **Skillのトリガー**
   Claude Codeで以下のキーワードを使用：
   ```
   "新しい...を開発してください"
   "...機能を実装してください"
   "このバグを修正してください"
   "開発計画を作成してください"
   "新しいプロジェクトを開始してください"
   ```

3. **インテリジェントアシスタンスを楽しむ**
   - 📋 開発計画の自動生成
   - 🏗️ プロジェクト構造の分析
   - 🔄 開発プロセスのガイド
   - 📊 進捗状態の同期

### Cursorでの使用

1. **Claude統合の設定**
   - CursorがClaude APIで設定されていることを確認
   - DevContinuum Skillドキュメントをプロジェクト知識ベースとして使用

2. **プロジェクトドキュメントの参照**
   ```
   プロジェクト構造についてはdocs/structure/project-analysis.mdを参照してください
   開発計画フォーマットについてはdocs/plan/example-user-auth.mdを参照してください
   ```

3. **組み合わせ使用**
   - DevContinuumが生成した計画を使用して開発をガイド
   - 標準化されたワークフロードキュメントを参照
   - プロジェクトコーディング規約に従う

### その他のAIツールでの使用

**一般的な方法**:
1. `docs/`ディレクトリをプロジェクトドキュメントライブラリとして使用
2. 開発前に関連ドキュメントを読んで規約を理解
3. `docs/plan/template.md`を使用して開発計画を作成
4. `docs/structure/development-workflows.md`のプロセスに従う

## 🎯 ベストプラクティス

### 1. 📝 計画優先
- **直接コーディングを開始しない**、まずDevContinuumに計画作成を依頼
- **タスク範囲を明確にする**、範囲蔓延を回避
- **リスクを特定する**、事前に対処策を準備

### 2. 🔄 小歩で走る
- **大タスクを小さく分解**、管理可能なサブタスクに
- **3-5個のサブタスク完了ごと**にレビューを実施
- **進捗をタイムリーに更新**、状態同期を維持

### 3. 📚 ドキュメントが王
- **すべての重要決定**をドキュメントに記録
- **プロジェクト構造ドキュメントを定期的に更新**、最新状態を維持
- **経験教訓を沈降**、チーム知識ベースを形成

### 4. 🛡️ 安全第一
- **重要操作前に**チェックポイントを作成
- **ロールバックメカニズムを有効化**、安全な後退を確保
- **セキュリティ規約に従う**、脆弱性の導入を回避

### 5. 🤝 チームコラボレーション
- **計画ドキュメントを共有**、チームのアライメントを確保
- **進捗を定期的に同期**、ブロッキングを早期発見
- **コード規約を統一**、コード一貫性を維持

## 🚀 開始方法

### 前提条件
- GitHubプロジェクト
- Claude Codeまたは互換性のあるAIプログラミングツール
- 基本的なGitと開発環境

### インストール手順
1. DevContinuum Skillをクローンまたはダウンロード
2. Skillファイルをプロジェクトの`.claude/skills/`ディレクトリに配置
3. プロジェクトドキュメント構造が規約に準拠していることを確認
4. 開始！

### 推奨設定
```yaml
# .claude/settings.json
{
  "default_skill": "DevContinuum",
  "auto_save": true,
  "checkpoint_interval": 3,
  "enable_external_skills": true
}
```
