# DevContinuum 快速入门指南 🚀

## 🎯 一句话介绍

**DevContinuum 是一个 AI 驱动的开发流程优化工具，专门解决大项目开发中的上下文丢失、流程混乱、进度跟踪困难等问题。**

## 🤔 你是否遇到过这些问题？

- 🤯 **"任务太大，写着写着就忘了最开始的设计..."
- 📝 **"每次都要重新解释项目结构和技术栈..."
- 🔄 **"开发到一半系统崩溃，进度全没了..."
- 👥 **"团队协作时代码规范不统一..."
- 📊 **"不知道项目进展到什么程度了..."

如果你有以上任何一个问题，DevContinuum 就是为你准备的！

## ✨ 它能做什么？

### 🧠 记住一切
- 自动保存开发计划、进度、决策到项目文档
- 即使 AI 会话中断，也能无缝恢复工作
- 关键信息永不丢失

### 📋 规范流程
- 提供标准化的开发工作流 (Feat/Fix/CI)
- 每一步都有明确指导，避免遗漏重要环节
- 代码审查和测试要求一目了然

### 📊 实时跟踪
- 任务清单式进度管理
- 实时同步开发状态
- 项目进展一目了然

### 🛡️ 容灾保护
- 自动创建检查点 (Checkpoint)
- 支持断点续传
- 异常情况自动处理

## 🚀 3分钟快速开始

### 第一步：安装

1. 将 DevContinuum 文件夹放入你的项目：
```bash
your-project/
└── .claude/
    └── skills/
        └── DevContinuum/    ← 放这里
```

2. 确保你的项目有 `docs/` 目录：
```bash
your-project/
└── docs/
    ├── plan/               ← 计划文档
    ├── feat/               ← 功能文档  
    ├── fix/                ← 修复文档
    └── structure/          ← 项目结构
```

### 第二步：使用

在 Claude Code 中这样说：

```
"帮我开发一个用户登录功能"
"制定一个API开发计划"  
"修复这个bug"
"开始一个新项目"
```

### 第三步：享受

DevContinuum 会自动：
- ✅ 生成详细的开发计划
- ✅ 分析项目结构和技术栈
- ✅ 指导你一步步完成开发
- ✅ 实时同步进度到文档
- ✅ 记录所有重要决策

## 🎪 举个栗子

### 场景：开发用户注册功能

**你说**:
```
"帮我开发一个用户注册功能，包括表单验证和API接口"
```

**DevContinuum 自动**:

1. 📋 **生成计划文档** `docs/plan/user-registration.md`:
   ```markdown
   # 用户注册功能开发计划
   
   ## 执行步骤
   - [ ] 1. 设计用户数据模型
   - [ ] 2. 创建注册表单组件
   - [ ] 3. 实现表单验证
   - [ ] 4. 开发注册API接口
   - [ ] 5. 编写单元测试
   - [ ] 6. 集成测试
   ```

2. 🏗️ **分析项目结构**:
   ```
   检测到项目使用 React + TypeScript + Express
   代码规范：ESLint + Prettier
   测试框架：Jest + React Testing Library
   ```

3. 🔧 **指导开发**:
   ```
   建议先创建用户数据模型：
   - 用户ID (UUID)
   - 用户名 (必填，3-20字符)
   - 邮箱 (必填，邮箱格式)
   - 密码 (必填，8位以上)
   - 创建时间
   ```

4. 📊 **同步进度**:
   ```markdown
   ## 进度状态
   - ✅ 数据模型设计完成
   - 🔄 表单组件开发中 (60%)
   - ⏳ API接口待开始
   - ⏳ 测试待开始
   ```

## 🎯 核心工作流

### 1. Feat 流程 (新功能开发)
```
需求 → 计划 → 设计 → 开发 → 测试 → 文档 → 完成
```

### 2. Fix 流程 (Bug修复)
```
问题 → 复现 → 分析 → 修复 → 验证 → 部署 → 关闭
```

### 3. CI 流程 (持续集成)
```
提交 → 检查 → 测试 → 构建 → 部署 → 监控
```

```

├── .claude/                           # Claude配置
│   └── skills/
│       └── DevContinuum/             # 核心Skill
├── docs/                             # 项目文档
│   ├── plan/                        # 任务计划
│   │   └── [任务名称].md            # 具体计划
│   ├── feat/                        # 功能文档
│   │   └── [功能名称]/              # 功能相关
│   ├── fix/                         # 修复文档
│   │   └── [修复名称]/              # 修复相关
│   └── structure/                   # 项目结构
│       ├── project-analysis.md      # 项目分析
│       └── development-workflows.md # 开发流程
└── src/                            # 源代码
    ├── components/                 # 组件
    ├── pages/                      # 页面
    └── utils/                      # 工具函数
```

## 💡 使用技巧

### 1. 计划先行 📋
- 不要直接开始编码
- 先让 DevContinuum 帮你制定计划
- 明确任务范围和验收标准

### 2. 小步快跑 🔄
- 将大任务拆解为小任务
- 每完成几个任务就回顾一次
- 及时更新进度状态

### 3. 文档为王 📚
- 重要决策都要记录
- 定期更新项目文档
- 形成团队知识库

### 4. 善用模板 📝
- 使用 `docs/plan/template.md` 制定计划
- 参考 `docs/structure/` 中的规范
- 遵循既定的工作流程

## 🚨 注意事项

### 1. 权限设置
确保 Claude Code 有足够的权限：
- 读取项目文件
- 创建和修改文档
- 执行必要的命令

### 2. 项目规范
- 遵循项目的代码规范
- 保持文档结构一致
- 定期同步项目状态

### 3. 备份重要
- 重要操作前创建检查点
- 定期备份关键文档
- 启用版本控制
